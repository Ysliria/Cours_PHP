Déclaration de variables

Règles de déclaration

Les variables en PHP sont représentées par le signe $ suivi d'un nom sensible à la casse. Un nom valide commence par une lettre ou un underscore, suivi de lettres, de chiffres et/ou d'underscores.

ATTENTION : $this est une variable de PHP qui ne peut pas être réassignée ! 

<?php
$une_variable;
$uneAutreVariable;
$_variable;

PSR : les conventions

Les PSR pour PHP Standards Recommandations, indique un certains nombre de conventions qui ont été définies pour uniformiser les pratiques en PHP.
Les PSR-1 et PSR-12, définissent respectivement les normes de codage de base et le style de codage étendu.
Si ces recommandations déterminent que les variables peuvent être définies en camelCase, UpperCamelCase et snake_case ; nous allons prendre le parti dès à présent de les écrire en camelCase.

Affectation

L'afectation d'une valeur à une variable se fait avec le signe =.
Si on décide de faire plusieurs affection à un même variable, la dernière affectation sera la valeur réelle de la variable. En effet chaque affectation écrase la valeur dela précédente affectation. 

<?php
$variableUne = 'Variable une';
$variable2 = 2;
$troisiemeVariable = '3';

La portée des variables

Les variables ont une portée lié au script dans lequel elles sont définies. Néanmoins, si elles sont définies dans une fonction, elles auront une portée locale à la fonction. 

<?php
$maVariable = 'ma variable';

function maFunction()
{
    $maVariable = 'une autre variable';
    echo $maVariable; // Affiche : une autre variable.
}

echo $maVariable; // Affiche : une variable.4
